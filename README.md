## ✨ Features

- ✅ Full WebAuthn **Registration (Attestation)** ceremony
- ✅ Full WebAuthn **Authentication (Assertion)** ceremony
- ✅ HTTPS deployment with self-signed certificate
- ✅ Mobile testing via **Cloudflare Tunnel** (free)
- ✅ Support for **Windows Hello**, **Android biometrics**, **Google Password Manager**
- ✅ Protected page showing username and full authenticator information
- ✅ **Signature counter** tracking (cloned authenticator detection)
- ✅ Server verifies **challenge, origin, RP ID hash, UP/UV flags, signature**
- ✅ Negative test: rejects foreign credentials and tampered signatures
- ✅ Cross-device credential usage via environment variable RP configuration (Bonus)

## 🔐 Generate TLS Certificate

WebAuthn requires HTTPS. Generate a self-signed certificate for local development.

### Windows — Open Git Bash

Right-click Desktop → **Git Bash Here**, then:

```bash
cd ~/Desktop/webauthn-project/webauthn-project
mkdir -p certs

openssl req -x509 -newkey rsa:4096 \
  -keyout certs/key.pem \
  -out certs/cert.pem \
  -days 365 -nodes \
  -addext "subjectAltName=DNS:localhost,IP:127.0.0.1"
```

Answer the prompts:
```
Country Name (2 letter code): BD
State or Province Name: Dhaka
Locality Name: Dhaka
Organization Name: CSE722
Organizational Unit Name: WebAuthn
Common Name: localhost
Email Address: (press Enter to skip)
```

### Linux / Kali — Open Terminal

```bash
cd ~/Desktop/webauthn-project/webauthn-project
mkdir -p certs

openssl req -x509 -newkey rsa:4096 \
  -keyout certs/key.pem \
  -out certs/cert.pem \
  -days 365 -nodes \
  -subj "/CN=localhost/O=CSE722/C=BD" \
  -addext "subjectAltName=DNS:localhost,IP:127.0.0.1"
```

### Verify certificates were created

```bash
ls certs/
# Should show: cert.pem  key.pem
```

## 📱 Mobile Browser Testing

### Method — Cloudflare Tunnel (Free, No Account Needed)

Cloudflare Tunnel provides a free public HTTPS URL for your local server, enabling testing from any mobile device.

#### Step 1 — Download cloudflared

**Windows:**
```cmd
mkdir C:\cloudflared
# Download cloudflared-windows-amd64.exe to C:\cloudflared\
# Rename to cloudflared.exe
```

Download from: https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-windows-amd64.exe

**Linux / Kali:**
```bash
wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64
chmod +x cloudflared-linux-amd64
sudo mv cloudflared-linux-amd64 /usr/local/bin/cloudflared
```

#### Step 2 — Start Server in HTTP Mode

**Windows:**
```cmd
cd server
call ..\venv\Scripts\activate
set TLS_CERT=notexist
set TLS_KEY=notexist
set RP_ID=PLACEHOLDER
set ORIGIN=PLACEHOLDER
python app.py
```

**Linux / Kali:**
```bash
cd server
source ../venv/bin/activate
python3 app.py
```

Server shows:
```
[!] TLS certificate not found – falling back to HTTP
* Running on http://127.0.0.1:5000
```

#### Step 3 — Start Cloudflare Tunnel (New Terminal)

**Windows:**
```cmd
C:\cloudflared\cloudflared.exe tunnel --url http://localhost:5000
```

**Linux / Kali:**
```bash
cloudflared tunnel --url http://localhost:5000
```

Wait 10-15 seconds. You will see:
```
+-----------------------------------------------------------------------+
| Your quick Tunnel has been created! Visit it at:                      |
| https://abc-def-123-xyz.trycloudflare.com                             |
+-----------------------------------------------------------------------+
```

#### Step 4 — Restart Server with Tunnel Domain

Copy the tunnel URL domain (e.g. `abc-def-123-xyz.trycloudflare.com`)

Stop the server (Ctrl+C) and restart:

**Windows:**
```cmd
set TLS_CERT=notexist
set TLS_KEY=notexist
set RP_ID=abc-def-123-xyz.trycloudflare.com
set ORIGIN=https://abc-def-123-xyz.trycloudflare.com
python app.py
```

**Linux / Kali:**
```bash
RP_ID=abc-def-123-xyz.trycloudflare.com \
ORIGIN=https://abc-def-123-xyz.trycloudflare.com \
python3 app.py
```

#### Step 5 — Test on Phone

Open each browser on your Android/iOS phone and go to:
```
https://abc-def-123-xyz.trycloudflare.com
```

Test with these 3 browsers:
- **Chrome Mobile** — username: `mobile_chrome`
- **Firefox Mobile** — username: `mobile_firefox`
- **Edge Mobile** — username: `mobile_edge`

Each browser will prompt for **fingerprint or PIN** when registering.

> 💡 The tunnel URL changes every time you restart cloudflared. Always update RP_ID and ORIGIN when you get a new URL.

## 🔌 API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/` | Serves the frontend SPA |
| `POST` | `/register/begin` | Generate registration options (challenge) |
| `POST` | `/register/complete` | Verify attestation and store credential |
| `POST` | `/authenticate/begin` | Generate authentication options (challenge) |
| `POST` | `/authenticate/complete` | Verify assertion and create session |
| `GET` | `/protected` | Protected resource (requires session) |
| `POST` | `/logout` | Clear session |

## 🔒 WebAuthn Flow

### Registration Flow

```
Browser                    Server
  |                           |
  |-- POST /register/begin -->|
  |<-- challenge + options ---|
  |                           |
  |-- navigator.credentials.create() --> Authenticator
  |<-- attestation object --------------|
  |                           |
  |-- POST /register/complete -->|
  |   (attestation response)    |
  |                           |-- Verify: challenge ✓
  |                           |-- Verify: origin ✓
  |                           |-- Verify: RP ID hash ✓
  |                           |-- Verify: UP flag ✓
  |                           |-- Verify: attestation ✓
  |                           |-- Store: public key + sign count
  |<-- { status: ok } --------|
```

### Authentication Flow

```
Browser                    Server
  |                           |
  |-- POST /authenticate/begin -->|
  |<-- challenge + allow list ----|
  |                           |
  |-- navigator.credentials.get() --> Authenticator
  |<-- assertion (signed challenge) --|
  |                           |
  |-- POST /authenticate/complete -->|
  |   (assertion response)         |
  |                           |-- Verify: challenge ✓
  |                           |-- Verify: origin ✓
  |                           |-- Verify: RP ID hash ✓
  |                           |-- Verify: UP flag ✓
  |                           |-- Verify: signature ✓
  |                           |-- Verify: sign count ✓
  |                           |-- Update: sign count
  |<-- { status: ok, username } --|
```

### Server Verification Steps (WebAuthn Spec)

| Ceremony | Step | What is Verified |
|---|---|---|
| Registration | §7.1 Step 7 | clientDataJSON.type = 'webauthn.create' |
| Registration | §7.1 Step 11 | Challenge matches server-issued challenge |
| Registration | §7.1 Step 13 | Origin matches expected origin |
| Registration | §7.1 Step 14 | RP ID hash in authenticatorData |
| Registration | §7.1 Step 15 | User-Present flag set |
| Authentication | §7.2 Step 7 | clientDataJSON.type = 'webauthn.get' |
| Authentication | §7.2 Step 11 | Challenge matches server-issued challenge |
| Authentication | §7.2 Step 13 | Origin matches expected origin |
| Authentication | §7.2 Step 14 | RP ID hash in authenticatorData |
| Authentication | §7.2 Step 15 | User-Present flag set |
| Authentication | §7.2 Step 20 | Signature verified with stored public key |
| Authentication | §7.2 Step 17 | Sign count > previously stored count |

## 🛠️ Troubleshooting

| Error | Cause | Fix |
|---|---|---|
| `ModuleNotFoundError: webauthn.helpers` | Wrong webauthn package installed | `pip uninstall webauthn && pip install webauthn==2.7.1` |
| `The operation is insecure` | Accessing WebAuthn over HTTP on non-localhost | Must use `https://` URL — generate TLS cert |
| `Certificate warning in browser` | Self-signed cert not trusted | Click Advanced → Proceed (Chrome) or Accept Risk (Firefox) |
| `ERR_EMPTY_RESPONSE` via tunnel | Server running HTTPS but tunnel expects HTTP | Set `TLS_CERT=notexist` to force HTTP mode |
| `RP ID not registrable domain suffix` | Using .dev or .ngrok-free.dev domain | Use Cloudflare Tunnel (.trycloudflare.com) instead |
| `Session expired` error | Session not persisting between /begin and /complete | Check SESSION_FILE_DIR path exists |
| `AttributeError: attestation_type` | Library version mismatch | Use `webauthn==2.7.1` exactly |
| `venv\Scripts\activate` fails | PowerShell execution policy | Run: `Set-ExecutionPolicy RemoteSigned` as admin |
| Port 5000 already in use | Another process using port 5000 | `set PORT=8443` and use `https://localhost:8443` |
| ngrok `.dev` domain rejected | HSTS preload list blocks WebAuthn | Switch to Cloudflare Tunnel |

## 🔬 Negative Tests

### Test 1 — Unregistered User
```
Sign In tab → username: "hacker" (never registered) → Sign In
Server response (HTTP 400): {"error": "No credentials registered for this user"}
```

### Test 2 — Tampered Signature
Open F12 → Console, paste before Sign In:
```javascript
const orig = navigator.credentials.get.bind(navigator.credentials);
navigator.credentials.get = async function(opts) {
  const r = await orig(opts);
  const sig = new Uint8Array(r.response.signature);
  sig[0] ^= 0xFF;
  return r;
};
```
```
Server response (HTTP 400): {"error": "Authentication failed: Signature verification failed"}
```

### Test 3 — Foreign Credential (Wrong User)
```
Register alice and bob separately
Sign in as bob but submit alice's credential ID
Server response (HTTP 403): {"error": "Credential does not belong to this user – access denied"}
```
